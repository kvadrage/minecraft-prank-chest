# v0.2 — Vanilla Chest Approach (Architecture Rewrite)

## Context for the Implementer

This spec was written by the design team (human + AI assistant). We've been through v0.1 (entity-based custom chest) and v0.1.1 (bug fixes) — both had 6+ critical bugs because recreating chest behavior from scratch is extremely hard in Bedrock.

**You (Claude Code) have implementation freedom** on all details not marked as MUST. Where we give recommendations, treat them as starting points — if you know a better approach, use it and leave a comment explaining why. If something is ambiguous or you see multiple valid approaches, pick the best one and document the tradeoff in a code comment.

**You cannot test in Minecraft.** The human will test on Bedrock `1.26.1301.0` (Windows). Write defensive code, log warnings for edge cases, and leave `// TODO: verify in-game` comments where you're uncertain about API behavior.

**Ask us (via comments in code or a QUESTIONS.md file) if:**
- An architectural decision feels wrong
- You find that an API we assumed exists doesn't actually work
- There are multiple valid approaches with significant tradeoffs

---

## Goal

Replace the entity-based Prank Chest with a **real vanilla `minecraft:chest`** plus Script API logic. The chest should be visually and behaviorally indistinguishable from a normal chest — because it IS a normal chest. We only add hidden downgrade logic.

---

## MUST (Non-negotiable)

These are architectural decisions we've validated through painful iteration. Don't change these.

1. **No custom block, no custom entity for the placed chest.** The placed chest MUST be a vanilla `minecraft:chest`. This is the whole point of v0.2.
2. **Custom item** — `prankchest:prank_chest` is a craftable item that, when placed, creates a vanilla chest marked as prank.
3. **Prank marking** — Script must track which chests are prank chests. The tracking must survive world reload.
4. **Downgrade logic** — Same tables as v0.1 (ores, ingots, tools, armor). Per-slot independent timers. Default delay: 1000 ticks (~50 real seconds). Items not in any chain are left unchanged.
5. **Breaking a prank chest** drops a `prankchest:prank_chest` item (not a normal chest) + all contents in original (non-downgraded) form.
6. **Recipe** must NOT conflict with any vanilla recipe. v0.1 conflicted with noteblock (8 planks + redstone).
7. **`@minecraft/server` stable API only** — no beta versions. Last known working: `1.16.0`. Use the latest stable you're confident works with Bedrock 1.21.50+.
8. **File cleanup** — Remove all entity-based files from v0.1.x that are no longer needed (custom block JSON, entity definitions, geometry, animations, render controllers, loot tables). Keep the repo clean.

## SHOULD (Strong recommendations, but you decide the details)

### Prank Registry

We think `world.setDynamicProperty()` with a JSON map keyed by coordinates (`"x,y,z,dimId"`) is the way to go, since vanilla blocks don't support `block.setDynamicProperty()`.

But consider:
- Is there a better storage approach? (scoreboard, tags on nearby entities, etc.)
- What's the actual size limit for world dynamic properties? Will it scale to 50-100 prank chests?
- How do you handle chunk unloading gracefully?

### Break Interception

We identified three approaches:
- **A:** `beforeEvents.playerBreakBlock` — cancel, manually clear inventory + destroy block + drop prank item
- **B:** `afterEvents.playerBreakBlock` — let vanilla drop happen, also drop prank item (player gets both — messy)
- **C:** `afterEvents.playerBreakBlock` — find and remove the dropped vanilla chest entity nearby, spawn prank item

**Analyze these and pick the best one.** If there's an Option D we missed, use it. Leave a comment explaining your choice.

### Explosion Handling

Prank chests should be cleaned from registry when destroyed by TNT/creeper. We're not sure of the best event to hook (`beforeEvents.explosion`? periodic registry cleanup? something else?). Your call.

### Chest Orientation

When placed, the chest should face the player (same as vanilla placement). Vanilla chests use the `minecraft:cardinal_direction` block state. Set it based on player yaw.

### Recipe

We suggest: planks + tripwire hook + redstone. Tripwire hook = "it's a trap!" thematic fit. But if you see a conflict or have a better idea, go for it. Just document it.

### Item Icon

The prank chest item should look like a vanilla chest in the inventory. Reference the vanilla chest texture if possible; copy it if you must.

## Downgrade Tables (reference)

### Ores
```
ancient_debris → diamond_ore
diamond_ore → iron_ore  
iron_ore → copper_ore
copper_ore → gold_ore
gold_ore → dirt
```
Deepslate variants follow the same chain. Nether gold ore → dirt.

### Ingots / Gems
```
netherite_ingot → diamond
diamond → iron_ingot
iron_ingot → copper_ingot
copper_ingot → gold_ingot
gold_ingot → dirt
```

### Tools (sword, pickaxe, hoe, shovel, axe)
```
netherite → diamond → iron → wooden → golden → dirt
```
Skip copper tier (doesn't exist in vanilla).

### Armor (helmet, chestplate, leggings, boots)
```
netherite → diamond → iron → leather → golden → dirt
```
Skip copper tier.

### Everything else
Left unchanged. No downgrade.

## Migration Note

v0.2 is a **breaking change** from v0.1.x. Existing prank chest entities will stop working. Users need to:
1. Run `/kill @e[type=prankchest:prank_chest]` to remove old entities
2. Remove and re-apply the addon

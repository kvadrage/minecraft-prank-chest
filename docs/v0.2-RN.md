# Prank Chest v0.2.0 — Release Notes

**Target:** Minecraft Bedrock Edition 1.21.50+ (tested on 1.26.1301.0)
**API:** `@minecraft/server` 1.16.0 (stable)
**Branch:** `feat/v0.2-vanilla-chest` — pending in-game testing before merge to main

---

## Architecture Rewrite — Vanilla Chest Approach

v0.1.x used a custom entity to simulate a chest. This caused 6+ critical bugs because
recreating all chest behavior (inventory UI, animations, sounds, orientation, explosions)
from scratch in Bedrock is extremely hard and fragile.

v0.2 replaces the entire approach: **the placed chest is a real `minecraft:chest`**. It
looks, sounds, and behaves identically to a vanilla chest because it IS one. The prank
logic is a hidden layer on top — not a replacement of the chest itself.

---

## What Changed

### Placed chest is now a real vanilla chest

No custom entity, no custom block in the world. When a player places a `prankchest:prank_chest`
item, the script intercepts `beforeEvents.itemUseOn`, cancels the default use, and places a
real `minecraft:chest` via `block.setPermutation()` with the correct `minecraft:cardinal_direction`
facing the player. From that point forward the chest is indistinguishable from any other chest.

### Prank registry (world dynamic property)

Prank chests are tracked in `world.getDynamicProperty("prankchest:registry")` — a JSON map
keyed by block coordinates + dimension ID (`"x,y,z,dimId"`). The registry persists across
world reloads. Only active timer slots are stored per entry, keeping the payload small.

### Break interception — drops prank_chest item, not normal chest

`beforeEvents.playerBreakBlock` cancels the vanilla break (which would drop a normal chest
item), then manually drops all inventory contents + one `prankchest:prank_chest` item and
removes the block. The player never receives a `minecraft:chest` item from breaking a prank
chest.

**Trade-off:** The player's tool does not consume durability on this break — the cancel fires
before vanilla durability processing. Minor cosmetic inconsistency, accepted for v0.2.

### Explosion handling

`beforeEvents.explosion` scans impacted blocks for registered prank chests and queues their
removal from the registry. A 5-minute periodic stale sweep also handles pistons, `/fill`,
and other block-removal paths that bypass `playerBreakBlock`.

TNT destroying a prank chest drops a normal chest item (vanilla loot) — the spec only
requires registry cleanup for explosions, not a special item drop.

### Custom item replaces custom block

`prankchest:prank_chest` is now defined as a custom **item** (`items/prank_chest.json`,
format 1.20.50) rather than a custom block. The item has no `minecraft:block_placer` —
placement is entirely script-driven. Icon comes from `item_texture.json` →
`textures/blocks/chest_front.png`.

## Files Removed

These v0.1.x files are no longer needed and have been deleted:

| Removed | Reason |
|---------|--------|
| `behavior_pack/blocks/prank_chest.json` | No custom block in world |
| `behavior_pack/entities/prank_chest.json` | No custom entity |
| `behavior_pack/loot_tables/` | Entity-based drop logic replaced by break handler |
| `resource_pack/animation_controllers/` | No entity animations |
| `resource_pack/animations/` | No entity animations |
| `resource_pack/entity/` | No client entity definition |
| `resource_pack/models/` | No custom geometry |
| `resource_pack/render_controllers/` | No entity rendering |
| `resource_pack/textures/terrain_texture.json` | No custom block material |
| `resource_pack/textures/entity/chest/normal.png` | Entity texture no longer used |

## Known Limitations

| Issue | Status |
|-------|--------|
| Chest facing direction may be inverted (faces away from player instead of toward) | TODO: verify in-game — one-line fix in `yawToCardinal()` if wrong |
| Item icon is a flat chest face (chest_front.png), not an isometric sprite | Cosmetic — acceptable for v0.2 |
| Tool durability not consumed when breaking a prank chest | Minor trade-off of Option A break interception |
| TNT drops a normal chest item, not a prank_chest item | By design — spec only requires registry cleanup for explosions |
| Dynamic property limit (~32 KB) caps registry at ~160 simultaneous prank chests | Warn logged at 32,000 chars; sufficient for normal play |
| `beforeEvents.explosion.getImpactedBlocks()` may not be available in stable 1.16.0 | Wrapped in try/catch; periodic cleanup is fallback |

See `QUESTIONS.md` for the full list of 8 runtime unknowns to verify in-game.

---

## Migration from v0.1.x

This is a **breaking change**. Existing prank chest entities will stop working.

Before applying v0.2:
1. Run `/kill @e[type=prankchest:prank_chest]` to remove all v0.1.x entities
2. Remove the old addon from the world
3. Apply v0.2 addon

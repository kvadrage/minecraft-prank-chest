# v0.1.1 — Bug Fix & Visual Parity Sprint

## Goal

Make the Prank Chest **indistinguishable** from a vanilla single chest in every way:
appearance, interaction, and behavior. Fix all bugs from v0.1 testing.

## Target Version

- Minecraft Bedrock Edition `1.21.50+` (tested on `1.26.1301.0`)
- `@minecraft/server` stable API — use version `1.16.0` (NOT beta)
- Addon namespace: `prankchest`

## Bugs to Fix (from v0.1 testing)

### BUG-1: "Unexpected version for the loaded data" on block JSON

**File:** `blocks/prank_chest.json`
**Problem:** `format_version: "1.21.40"` (and `"1.21.0"` before that) is not accepted by the game.
**Fix:** Use a known working block format_version. Try `"1.20.80"` or `"1.20.60"`. Check the Bedrock wiki for valid block format versions:
https://wiki.bedrock.dev/guide/format-version

### BUG-2: Entity renders semi-transparent / broken UV mapping

**Problem:** The chest entity has a polished look from one side but is semi-transparent from the other. The custom geometry (`geometry.prank_chest`) UV mapping doesn't match the vanilla chest texture (`textures/entity/chest/normal`).
**Root cause:** The vanilla chest texture layout is different from what the custom geo model assumes. The UV coordinates in `models/entity/prank_chest.geo.json` are wrong.
**Fix options (in order of preference):**
1. **Best:** Use the vanilla chest geometry identifier if possible (`geometry.chest` or `geometry.chest.small`) — zero UV mismatch risk
2. If that's not available, study the actual vanilla chest texture (`textures/entity/chest/normal.png`, 64×64) and fix UV coordinates to match exactly
3. Material `entity_alphatest` is correct for chest rendering (handles transparency in texture)

### BUG-3: Chest doesn't open on right-click

**Problem:** Right-clicking the entity does nothing — no inventory screen opens.
**Root cause:** `minecraft:inventory` component with `container_type: "chest"` alone does NOT make the entity interactable. The entity needs an explicit way to respond to player interaction.
**Fix:** The entity JSON needs an `minecraft:interact` component that triggers container opening. Research how vanilla chest entities and chest_minecart handle this. Possible approaches:
- Add `minecraft:interact` with `"interact_text": "action.interact.opencontainer"` or similar
- Use Script API `world.beforeEvents.playerInteractWithEntity` to intercept interaction and open the container programmatically (if that API exists for containers)
- Study `chest_minecart` entity JSON — it has a working chest inventory on interact

### BUG-4: Recipe conflicts with Noteblock

**Problem:** 8 planks + redstone = noteblock (vanilla recipe). Our recipe is identical.
**Logs:** `Recipe "minecraft:noteblock" has the same ingredients as prankchest:prank_chest's recipe`
**Fix:** Change crafting recipe. New recipe suggestion:
```
[Plank]  [Tripwire Hook] [Plank]
[Plank]  [Redstone Dust] [Plank]
[Plank]  [Plank]         [Plank]
```
Tripwire hook thematically fits (it's a trap!). Or use another combination that doesn't conflict with vanilla.

## New Requirements for v0.1.1

### REQ-1: Identical inventory icon

The Prank Chest item in the player's hotbar/inventory must look exactly like a vanilla chest. Options:
- Reference the vanilla chest item texture/icon directly
- Create a pixel-perfect copy in `textures/items/prank_chest.png` if referencing isn't possible
- Wire it up via `minecraft:icon` in block JSON or item JSON

### REQ-2: Identical placed appearance

When placed in the world, the Prank Chest entity must:
- Look exactly like a vanilla single chest (same size, same texture, same orientation)
- Face the player when placed (same rotation behavior as vanilla chest)
- NOT have visual glitches (no transparency, no Z-fighting, no flickering)

### REQ-3: Open/close behavior identical to vanilla chest

- Right-click opens the 27-slot chest UI
- Chest lid animates open when a player is viewing the inventory
- Chest lid closes when no player is viewing
- Opening/closing sounds match vanilla chest sounds

### REQ-4: Breaking behavior identical to vanilla chest

- Breaking the Prank Chest drops its contents (original items, not downgraded — timers cancel on break)
- Breaking drops the Prank Chest block item itself
- **Explosions** (TNT, creeper) should also destroy the chest and scatter contents
- Proper break time matching vanilla chest (axe speeds it up)

### REQ-5: Addon pack icon

Create a `pack_icon.png` (at least 256×256, ideally 512×512) for both behavior and resource packs. Design:
- A chest with a subtle "prank" element — e.g. a chest with one slightly glowing red eye, or a chest with a small redstone particle, or a chest with a mischievous grin
- Keep it recognizable as a chest at small sizes
- Save as `src/behavior_pack/pack_icon.png` and `src/resource_pack/pack_icon.png`

### REQ-6: Minimal block placeholder flash

When the player places the block, there's a visible flash of the placeholder block before the entity spawns. Minimize or eliminate this:
- Make the placeholder block invisible/transparent if possible
- Or reduce the spawn delay to absolute minimum
- Or explore alternatives (direct entity placement without block intermediary)

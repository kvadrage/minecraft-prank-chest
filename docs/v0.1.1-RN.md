# Prank Chest v0.1.1 — Release Notes

**Target:** Minecraft Bedrock Edition 1.21.50+  
**API:** `@minecraft/server` 1.16.0 (stable)

---

## Bug Fixes

### BUG-1: Block format_version error on world load
`blocks/prank_chest.json` used `"1.21.40"` which Bedrock rejected with "Unexpected version for the loaded data". Changed to `"1.20.80"`.

### BUG-2: Semi-transparent entity / broken UV mapping
The custom chest geometry had an incorrect UV coordinate for the lid's west face (`v=5` → `v=14`). Primary fix: the entity now references `geometry.chest` (vanilla built-in geometry), which carries zero UV mismatch risk. The custom geometry file (`models/entity/prank_chest.geo.json`) remains as a fallback with the UV corrected.

### BUG-3: Right-clicking the chest did nothing
`minecraft:inventory` alone does not open a container UI for custom entities. Added `minecraft:interact` with `interact_text: "action.interact.opencontainer"` to the entity definition. Both components together are required to trigger the chest screen.

### BUG-4: Recipe conflict with Noteblock
8 planks + redstone in the center is the vanilla Noteblock recipe. New recipe:

```
[ Plank ] [ Tripwire Hook ] [ Plank ]
[ Plank ] [ Redstone Dust ] [ Plank ]
[ Plank ] [    Plank      ] [ Plank ]
```

Tripwire hook is thematically fitting — it's a trap.

## New Features

### Chest faces the player on placement (REQ-2)
The spawned entity now rotates to face toward the player who placed it, matching vanilla chest orientation behavior.

### Lid open/close animation (REQ-3)
The chest lid animates open when a player interacts with it and closes when no player is within range. Driven by a 4-state animation controller (`closed → opening → open → closing`) tied to the `prankchest:is_open` entity property.

### Open/close sounds (REQ-3)
Plays `mob.chest.open` on right-click and `mob.chest.close` when the last nearby player moves away — matching vanilla chest audio.

### No block flash on placement (REQ-6)
Placement now intercepts `world.beforeEvents.itemUseOn` and cancels it before any block touches the world. The entity is spawned directly in the same tick via `system.run()`, eliminating the brief placeholder-block flash present in v0.1.

### Explosion support (REQ-4)
TNT and creeper explosions damage the entity, triggering `entityDie` which scatters all stored items and drops the Prank Chest item — same as breaking it by hand.

## Known Limitations

| Issue | Workaround |
|-------|------------|
| Inventory icon shows as oak planks, not a chest | Add `textures/items/prank_chest.png` + wire `minecraft:icon` in block JSON |
| No pack icon in the addon list | Add 256×256+ `pack_icon.png` to both pack roots |
| Lid close timing is approximate (proximity-based, not UI-state-based) | None in stable 1.16.0 API; exact detection requires a future API addition |
| `geometry.chest` may not resolve on all 1.21.x builds | Change `geometry.default` in `entity/prank_chest.entity.json` to `"geometry.prank_chest"` to use the included fallback geometry |
